Why teams choose Kastor

Domain-first RDF

Pure Kotlin domain APIs when you want them; RDF stays explicit via handles and generated vocabularies.

One API, many engines

Same patterns over memory stores, Jena (TDB2, etc.), RDF4J, and remote endpoints—swap with configuration, not rewrites.

Validation & quality

Native SHACL 1.2 Core engine, RDF4J bridge, and onto-quality catalogues for OWL, SKOS, and data-shape checks.

Kastor Gen

KSP + Gradle plugin: SHACL / JSON-LD–driven Kotlin interfaces and wrappers for maintainable ontology-bound code.
