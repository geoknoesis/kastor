# Support

## Usage questions

Use **[GitHub Discussions](https://github.com/geoknoesis/kastor/discussions)** for how-to questions, design feedback, and help interpreting RDF or Kastor APIs. That keeps the issue tracker focused on bugs and feature work.

**Published documentation** (when [GitHub Pages](https://docs.github.com/en/pages) is enabled with the Actions workflow): `https://<owner>.github.io/<repo>/` — source in the repo [`docs/`](docs/) folder.

## Bugs and regressions

Open a **[Bug report](https://github.com/geoknoesis/kastor/issues/new/choose)** (issue template) with a minimal reproducer and environment details.

## Security

Do **not** open a public issue for security vulnerabilities. Follow **[SECURITY.md](SECURITY.md)**.

## Releases and compatibility

- **0.x:** APIs may change; see **[CHANGELOG.md](CHANGELOG.md)** and release notes when upgrading.
- **Sponsoring / enterprise:** see the root **[README.md](README.md)** sponsorship section.

## Maintenance expectations

Kastor is maintained in the open; response times depend on maintainer availability. Pull requests with tests and clear descriptions are the fastest path to improvements.
